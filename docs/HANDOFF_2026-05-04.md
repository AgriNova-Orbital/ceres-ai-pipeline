## Project Purpose

Ceres AI Pipeline is an AgriNova-Orbital wheat rust risk prediction platform using Sentinel-2 time-series data, Google Earth Engine/Drive ingestion, staged GeoTIFF-to-NPZ dataset generation, CNN-LSTM training, threshold-oriented evaluation, and a WebUI control panel.

Primary user flow:

1. Export/download weekly Sentinel-2 risk rasters.
2. Normalize and inventory raw GeoTIFFs.
3. Build staged datasets at L1/L2/L4.
4. Run staged training matrix experiments.
5. Evaluate checkpoints and select best model/threshold.
6. Monitor and operate through the WebUI or CLI.

## Main Runtime Architecture

- `frontend/`: Next.js frontend, App Router, Tailwind CSS, default external dev port `3002`
- `apps/`: Flask/Gunicorn API and WebUI layer, default API/WebUI port `5055`
- `modules/jobs/`: RQ worker and background task definitions
- `modules/services/`: service-layer wrappers for dataset build, inventory, training matrix, evaluation
- `scripts/`: CLI entrypoints for direct operational execution and debugging
- `state/`: SQLite runtime state, ignored by git
- `data/`: raw and staged data, ignored by git
- `runs/`: training/evaluation outputs, ignored by git
- `reports/`: inventory/evaluation reports, ignored by git

Main request flow:

1. Frontend submits JSON to Flask API.
2. Flask validates input and enqueues RQ jobs.
3. Redis stores queue and job metadata.
4. Worker executes tasks and updates `job.meta`.
5. Frontend polls `/api/jobs` and `/api/jobs/<id>` for status.
